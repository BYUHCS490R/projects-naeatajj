Engagement - Tables

Questions & Answers

1. When do you use a table?

You use a table when you need to organize data into rows and columns. Tables are best used when information needs to be compared in a structured grid format, such as schedules, pricing charts, or statistical data.

2. When do you not use a table?

Do not use a table for website layout or general positioning. Flexbox and Grid are better for webpage layout because they are responsive and designed for modern web design.

3. When do you need to do extra work for table accessibility?

Extra accessibility work is needed when a table contains large or complex data. You should include a caption, header cells (<th>), and scope attributes so screen readers can interpret the data correctly.

The border attribute is outdated (HTML styling from the 1990s). Instead, modern web design uses CSS for styling because it separates structure (HTML) from design (CSS) and provides more flexibility and control.

5. What is the easiest way to color every other row a different color?

The easiest way is using CSS with tr:nth-child(even), which automatically applies a background color to every other table row.
